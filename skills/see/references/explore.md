# Explore

Read this file after the root skill routes the task to `explore`.

The goal is to build a reliable topic map, not to force a premature conclusion. See the full landscape first, then separate consensus from live disagreement.

## Core Flow

1. Cast a wide enough first pass to map the topic.
   Identify the main subtopics, main actors, representative sources, and common dispute points.
2. Find authoritative surveys, official material, and representative discussion.
   Use the authoritative material to define the backbone, then use representative discussion to reveal practical disagreement, variation, and unknowns.
3. Identify the mainstream views.
   Distinguish positions that are supported across multiple higher-tier sources from positions repeated mainly by one loud source.
4. Identify the major disagreements.
   Name whether the disagreement is about definitions, evidence, use cases, timelines, or something else.
5. Mark the still-open questions.
   Do not disguise uncertainty as consensus.
6. Separate established consensus from active debate.
   That separation is the main structure of a good exploratory answer.

## Precision Levels

### Standard mode (`standard`)

- Cover the mainstream views and representative sources.
- Give the user a clear topic map, the main directions, and the core disputes.
- Avoid overspending time on edge details.

### Strict mode (`strict`)

- Validate the key judgments with stronger sourcing.
- Add the main contrary views or important exception cases.
- Check whether a claim that looks mainstream is truly supported by multiple independent sources.

### Adversarial mode (`adversarial`)

- Challenge claims that look mainstream.
- Check whether the apparent consensus is mostly a loud-source illusion.
- Pay attention to quieter but higher-quality dissent and counterexamples.
- When a claim looks too smooth, deliberately look for its failure conditions.

## Answer-Specific Rules

- Separate mainstream views, minority views, and unknowns.
- If the topic is moving quickly, make the time boundary explicit.
- Do not write exploratory findings as if they were fully settled facts.
- If the user mainly wants orientation, prefer a structured map over a pile of links.

## Stop Conditions

Stop when all of these are true:

- the topic map is clear enough
- the mainstream views, major disagreements, and unknowns are separated
- new material mostly repeats existing camps rather than adding a new decisive angle
- the answer is already sufficient for the user to choose whether to go deeper
